# API Reference

SuperDocs provides specialized components for documenting REST APIs. This page demonstrates how to use them to create beautiful, interactive API documentation.

---

## APIEndpoint

Display an API endpoint with method badges and descriptions.

```jsx
<APIEndpoint method="GET" path="/users/{id}" description="Retrieve a user by their unique ID." />
```

<APIEndpoint method="GET" path="/users/{id}" description="Retrieve a user by their unique ID." />

### Supported Methods

<APIEndpoint method="GET" path="/users" description="List all users" />
<APIEndpoint method="POST" path="/users" description="Create a new user" />
<APIEndpoint method="PUT" path="/users/{id}" description="Update a user" />
<APIEndpoint method="PATCH" path="/users/{id}" description="Partially update a user" />
<APIEndpoint method="DELETE" path="/users/{id}" description="Delete a user" />

### Deprecated Endpoint

```jsx
<APIEndpoint method="GET" path="/v1/users" description="Old endpoint" deprecated />
```

<APIEndpoint method="GET" path="/v1/users" description="This endpoint is deprecated. Use /users instead." deprecated />

| Prop | Type | Description |
| :--- | :--- | :--- |
| `method` | `string` | HTTP method: `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `OPTIONS`, `HEAD` |
| `path` | `string` | The API path with placeholders like `{id}` |
| `description` | `string` | Brief description of the endpoint |
| `deprecated` | `boolean` | Mark endpoint as deprecated |

## ParamTable & Param

Document request parameters in a structured table.

```jsx
<ParamTable title="Path Parameters">
  <Param name="id" type="string" required>
    The unique identifier of the user.
  </Param>
</ParamTable>

<ParamTable title="Query Parameters">
  <Param name="limit" type="integer" default="10">
    Maximum number of results to return.
  </Param>
  <Param name="offset" type="integer" default="0">
    Number of results to skip.
  </Param>
</ParamTable>
```

<ParamTable title="Path Parameters">
  <Param name="id" type="string" required>
    The unique identifier of the user.
  </Param>
</ParamTable>

<ParamTable title="Query Parameters">
  <Param name="limit" type="integer" default="10">
    Maximum number of results to return.
  </Param>
  <Param name="offset" type="integer" default="0">
    Number of results to skip.
  </Param>
</ParamTable>

| Component | Prop | Type | Description |
| :--- | :--- | :--- | :--- |
| `ParamTable` | `title` | `string` | Section title (default: "Parameters") |
| `Param` | `name` | `string` | Parameter name |
| `Param` | `type` | `string` | Data type |
| `Param` | `required` | `boolean` | Whether the parameter is required |
| `Param` | `default` | `string` | Default value |

## ResponseField

Document API response fields.

### Example Response

```jsx
<ResponseField name="id" type="string">
  The user's unique identifier.
</ResponseField>
<ResponseField name="email" type="string">
  The user's email address.
</ResponseField>
<ResponseField name="name" type="string" nullable>
  The user's display name (may be null).
</ResponseField>
<ResponseField name="created_at" type="datetime">
  ISO 8601 timestamp of when the user was created.
</ResponseField>
```

## APIPlayground

Interactive API testing component that lets users make real API requests.

```jsx
<APIPlayground method="GET" path="/users" baseUrl="https://jsonplaceholder.typicode.com" />
```

<APIPlayground method="GET" path="/users" baseUrl="https://jsonplaceholder.typicode.com" />

| Prop | Type | Description |
| :--- | :--- | :--- |
| `method` | `string` | HTTP method |
| `path` | `string` | API endpoint path |
| `baseUrl` | `string` | Base URL for the API (editable by users) |

> [!TIP]
> The APIPlayground allows users to add custom headers, query parameters, and request bodies. Try it above!

## Complete API Documentation Example

Here's a full example of documenting an API endpoint:

### Get User by ID

<APIEndpoint method="GET" path="/users/{id}" description="Retrieve detailed information about a specific user." />

<ParamTable title="Path Parameters">
  <Param name="id" type="string" required>
    The unique identifier of the user (UUID format).
  </Param>
</ParamTable>

<ParamTable title="Query Parameters">
  <Param name="include" type="string">
    Comma-separated list of related resources to include. Options: `posts`, `comments`, `profile`.
  </Param>
</ParamTable>

#### Response

```json
{
  "id": "usr_abc123",
  "email": "john@example.com",
  "name": "John Doe",
  "avatar_url": "https://example.com/avatar.jpg",
  "created_at": "2024-01-15T10:30:00Z",
  "updated_at": "2024-06-20T14:45:00Z"
}
```

<ResponseField name="id" type="string">
  Unique user identifier with `usr_` prefix.
</ResponseField>
<ResponseField name="email" type="string">
  User's verified email address.
</ResponseField>
<ResponseField name="name" type="string" nullable>
  User's display name.
</ResponseField>
<ResponseField name="avatar_url" type="string" nullable>
  URL to the user's profile picture.
</ResponseField>
<ResponseField name="created_at" type="datetime">
  When the user account was created.
</ResponseField>
<ResponseField name="updated_at" type="datetime">
  When the user was last modified.
</ResponseField>

### Create User

<APIEndpoint method="POST" path="/users" description="Create a new user account." />

<ParamTable title="Request Body">
  <Param name="email" type="string" required>
    A valid email address. Must be unique.
  </Param>
  <Param name="password" type="string" required>
    Password (minimum 8 characters).
  </Param>
  <Param name="name" type="string">
    Optional display name.
  </Param>
</ParamTable>

#### Example Request

```bash
curl -X POST https://api.example.com/users \
  -H "Content-Type: application/json" \
  -d '{"email": "john@example.com", "password": "securepass123", "name": "John Doe"}'
```

#### Try It

<APIPlayground method="POST" path="/posts" baseUrl="https://jsonplaceholder.typicode.com" />

### Delete User

<APIEndpoint method="DELETE" path="/users/{id}" description="Permanently delete a user account." />

<Callout type="danger" title="Irreversible Action">
  Deleting a user will permanently remove all associated data including posts, comments, and files. This action cannot be undone.
</Callout>

<ParamTable title="Path Parameters">
  <Param name="id" type="string" required>
    The unique identifier of the user to delete.
  </Param>
</ParamTable>

#### Response

Returns `204 No Content` on success.
